02 · Part 1

User management

Every workspace has one company tree: a CPO at the top, directors under them, managers under directors, buyers under managers. Suppliers stay outside, invited per project. Six roles, one hierarchy.

First, a workspace
One client company = one workspace

A workspace is the isolated Crown environment of a single client company (Kiabi, Decathlon, etc.). It holds that client's users, org chart, projects, and address book of suppliers — and nothing else. Users from one workspace can never see another workspace's data.

Inside
Users & org chart
All buyer-side roles for this client.
Inside

The 6 roles

Super Admin
weight 100 · cross-tenant
Crown internal
Us. Can see every company, switch tenants, impersonate. Outside any client org chart.
Admin
weight 80 · client root
Client admin / CPO
Top of the client tree. Invites everyone else. Sees all projects and all users in their company.
Director
weight 60
Category director
Owns a category (Direct or Indirect). Sees all projects under them, runs steering on managers.
Manager
weight 40
Team lead
Runs a buyer team. Creates and runs projects, approves before they go live.
Buyer
weight 20
Operational buyer
Creates and runs their own projects end to end. Sees their own projects + anything shared with them.
Supplier
weight 10 · external
Bidding company
Outside the buyer tree. Sees only projects they're invited to, only their own bids, only their own award letters.
Hierarchy rule
A role can only invite or manage someone with a strictly lower weight.
Director can invite Manager & Buyer. Manager can invite Buyer. Buyer cannot invite anyone.

Two collaborator-level roles (per project)
Organizer · Viewer
On top of the company role, any user can be added to a specific project as Organizer (edit) or Viewer (read-only). Lets a Director jump into a Buyer's project without owning it.
"View as" (Super Admin & Admin)
Impersonate from the header
Super Admin can view as any user across any tenant. Admin can view as any user inside their own client company — to debug "what does this Buyer / Manager actually see?" without logging out. A banner stays on screen while active.

07

The 5 event types

Each event is a step in the project. Pick a type, configure it, run it, close it. A project can have any number of events in any order — RFX → Analysis → eAuction → Award → Contract is just the typical flow. You can also start a project directly with RFX, Analysis, or eAuction. Award and Contract are decision steps — they depend on an upstream event.

From PROJECT_EVENT_TYPES
